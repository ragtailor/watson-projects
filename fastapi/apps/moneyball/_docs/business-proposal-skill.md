# Proposal Generator

## Overview
이 스킬은 고객용 비즈니스 제안서를 작성한다.
부족한 정보가 있으면 먼저 질문하고, 임의로 가격이나 일정을 만들어내지 않는다.

## Workflow
1. 고객명, 프로젝트 범위, 산출물, 일정, 예산 정보를 확인한다.
2. 정보가 부족하면 제안서를 작성하기 전에 질문한다.
3. 요약, 문제 이해, 제안 솔루션, 산출물, 일정, 투자 비용, 다음 단계를 포함한다.
4. Output Format을 적용한다.
5. Quality Checklist를 확인한 뒤 최종 결과를 제공한다.

## Output Format
- 문서 형식: Markdown
- 길이: 500~800단어
- 주요 섹션: H2
- 톤: 전문적이고 명확하며 과장하지 않음
- 금지: 임의 가격 생성, 상투적 인사말, 사용자가 요청하지 않은 추가 섹션

## Quality Checklist
- 고객의 문제를 구체적으로 반영했는가?
- 산출물이 명확한가?
- 일정과 가격을 임의로 만들지 않았는가?
- 다음 행동이 분명한가?

## Examples
### 최소 입력
입력: "웹사이트 리뉴얼 제안서 써줘."
예상 동작: 고객명, 범위, 일정, 예산을 먼저 질문한다.

### 가격 미정
입력: "고객용 제안서인데 가격은 아직 정하지 않았어."
예상 동작: 가격 섹션에 "[가격 협의 중 -- 발송 전 확인 필요]"라고 표시한다.
