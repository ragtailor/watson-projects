# Benchmark Comparison — P5 (숨은 약점 적합성)

- eval name: p5-hidden-weakness
- winner: with-harness (하네스 포장) / 핵심 판단은 tie
- verdict: 게이트는 "이 업무는 적합"으로 판정했다(반복성이 실제로 충분하므로 옳은 판정). 숨겨둔 약점(인사이트 품질의 주관성)을 **"못 만든다"로 막은 게 아니라, 검증을 출처 추적(노트 근거)으로 재정의해 해결**했다. **baseline도 동일하게** 추출(사실)/작성(해석) 분리 + fidelity-reviewer + 근거 추적표 + 사람 게이트로 같은 해법에 독립 도달. 즉 숨은 약점은 강한 모델이 양쪽 다 *설계로 우회* 가능했다.

## 핵심 결론 (정직)
1. **게이트는 "못 만든다"를 남발하지 않는다** — 반복성이 실제 있는 업무는 적합 판정이 옳다. 숨은 검증 약점도 출처 추적·사람 게이트로 *해결 가능*하면 막지 않는다. 게이트의 가치는 "검증을 무엇에 기댈지"를 **강제로 surface**시키는 것(양쪽 다 그 질문에 답함).
2. **차별력은 또다시 포장(A4)** — 패턴·검증 분리·사람 게이트는 baseline도 안다. 재실행 가능한 상태/산출물 구조가 유일한 일관된 차별점.
3. 이 결과는 iteration 2 직후 넣은 강조 이동("가치는 패턴이 아니라 재실행 가능한 포장")을 **실측으로 재확인**한다.

## 한계
"못 만든다" 경로를 진짜로 트리거하려면, 반복성조차 약하고 출처 추적으로도 못 구하는 업무여야 하는데, 그런 업무는 강한 모델이 양쪽 다 알아챈다. 즉 게이트는 *강한 모델이 놓치는 것을 잡는 장치가 아니라*, 판단을 매번 체계적·명시적으로 강제하는 장치다. 약한 모델·신입 사용자·경계가 흐린 사례에서 가치가 더 크다.
